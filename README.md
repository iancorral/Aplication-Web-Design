# Que es Markdown?

Markdown es un lenguaje de marcado ligero que permite aplicar formato a un texto plano mediante símbolos específicos. Es el estándar para documentación técnica debido a su legibilidad.

## 1. Encabezados (Estructura)
Se utilizan almohadillas `#` al inicio de una línea. El número de símbolos determina la jerarquía.
- `# Título`: Define el título principal (H1).
- `## Subtítulo`: Define secciones (H2).
- `###`: Se utiliza para subsecciones (H3).

## 2. Formato de Texto (Énfasis)
Permiten resaltar palabras clave dentro de un párrafo:
- **Negrita**: Se logra encerrando el texto entre doble asterisco `**texto**`.
- *Cursiva*: Se logra con un solo asterisco `*texto*`.
- ~~Tachado~~: Se utiliza doble virgulilla `~~texto~~` para indicar contenido obsoleto.

## 3. Listas (Organización)
- **Desordenadas**: Se usa un guion `-` o asterisco `*` al inicio. Ideal para viñetas.
- **Ordenadas**: Se antepone el número seguido de un punto `1.`. Git y otros editores mantienen el orden automáticamente.

## 4. Enlaces y Recursos
- **Hipervínculos**: La sintaxis es `[Texto](URL)`. Al hacer clic en el texto, redirige al enlace.
- **Imágenes**: Es igual al enlace pero con un signo de exclamación al inicio: `![Alt](URL)`.

## 5. Código (Documentación Técnica)
Es la parte más vital para programadores:
- **Código en línea**: Se encierra entre acentos graves (\`) para resaltar comandos o variables como `git push`.
- **Bloques de código**: Se usan tres acentos graves (\` \` \`) seguidos del nombre del lenguaje para activar el resaltado de sintaxis (syntax highlighting).

## 6. Elementos de Apoyo
- **Citas**: Se usa el símbolo `>` para bloques de texto que son referencias de otras fuentes.
- **Separadores**: Tres guiones `---` crean una línea horizontal para dividir temas.
- **Tareas**: Se usan corchetes `[ ]` para pendientes y `[x]` para tareas completadas.
